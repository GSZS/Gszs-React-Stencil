* 2019/11/12

  ```js
  // 处理使用哈希路由导致的出现ParseError: Unrecognised input in theme.less on line 1, column 1:

  1: 配置Nginx的try_files $uri $uri/ /index.html; // 这句话表示没有匹配到路径就重定向回'/index.html'
  2: 设置homepage : '/'; // 这样设置是绝对路径，全都给我滚回Nginx中的index.html
  ```

  ```js
  /* 关于theme.less的问题 */
  <link rel="stylesheet/less" href="./theme.less" />
  一开始theme.less的获取方式是上面的那种，然后发现使用"BrowserRouter"路由方式挂到Nginx后不能获取报如下错误"in theme.less on line 1, column 1:"
  经过一番思考后是因为设置了"homepage: '/' "，然后"theme.less"被解析成了"index.html"所以报错

  // 解决方法
  将less文件改为css文件(因为将less文件上传到阿里云OSS然后打开链接会变成直接下载)。上传到阿里云OSS，获取到下载链接然后放入href中
  <link rel="stylesheet/less" href="https://gszs-static.oss-cn-shenzhen.aliyuncs.com/theme.css" />  
  
  // OK然后试着进入子页面后刷新，发现不会有任何问题了。搞定收工！
  ```
